## NOT RECOMMENDED FOR < 1 GB DRIVES

NEXISCORE GUI is a small Rust desktop file manager built with `eframe`/`egui`.
It opens a protected `environment/` workspace, lets you create folders and text
files, preview file contents, and delete files or directories with confirmation.

The app is intended to manage files inside `environment/` only. The **Back**
button is blocked at the environment root so the file browser cannot move up
into the project source code.

## Features

- Desktop GUI built with Rust, `eframe`, and `egui`
- Creates an `environment/` folder on startup
- Lists files and folders in the current environment directory
- Opens folders and navigates back within `environment/`
- Creates folders with **Make Directory**
- Creates or overwrites text files with **Write File**
- Shows text file contents in the preview editor
- Deletes files and folders through a confirmation step
- Loads basic settings from `config.lst`
- Uses a custom font from `assets/font.ttf`
- Embeds the Windows executable icon from `assets/logo.png`

## Repository Contents

```text
.
|-- Cargo.toml
|-- Cargo.lock
|-- build.rs
|-- README.md
|-- assets/
|   |-- font.ttf
|   `-- logo.png
`-- src/
    `-- main.rs
```

The `target/` folder is not committed to GitHub. It is generated by Cargo when
you build or run the project.

`assets/logo.png` is used as the Windows executable icon. During Windows builds,
`build.rs` converts this PNG into a temporary `.ico` file and embeds it into the
compiled `.exe`.

## Requirements

Install Rust before compiling:

- Rust toolchain: <https://www.rust-lang.org/tools/install>
- Cargo, included with Rust

On Windows, the stable MSVC Rust toolchain is recommended. If Rust is installed
through `rustup`, this project should build with the default stable toolchain.

Check your installation:

```powershell
rustc --version
cargo --version
```

## Compile From Source

Clone the repository:

```powershell
git clone https://github.com/zeerak587-cloud/NexisCoreTerminal/
cd NexisCoreTerminal
```

Build the debug executable:

```powershell
cargo build
```

After building, Cargo creates:

```text
target/debug/nexiscore_gui.exe
```

Build an optimized release executable:

```powershell
cargo build --release
```

After a release build, Cargo creates:

```text
target/release/nexiscore_gui.exe
```

## Run The App

The easiest development command is:

```powershell
cargo run
```

Run this command from the repository root. The app creates and uses:

```text
environment/
```

inside the current working directory.

You can also run a built executable directly:

```powershell
.\target\debug\nexiscore_gui.exe
```

Important: the app creates `environment/` relative to the folder it is launched
from. For the cleanest behavior, run `cargo run` from the repository root.

## How To Use

### Create A Folder

1. Type a folder name in **New name**.
2. Click **Make Directory**.
3. The new folder appears in the left file list.

### Create A File

1. Type a filename in **New name**.
2. Type the file contents in **New file content**.
3. Click **Write File**.

If a file with that name already exists, **Write File** overwrites it.

### Open A Folder

1. Select a `[DIR]` entry in the left file list.
2. Click **Open Directory**.

### Go Back

Click **Back** to move up one folder.

The app will not go above `environment/`. If you are already at the environment
root, it shows:

```text
Already at environment root
```

### Preview A File

Select a file in the left file list. The app loads its text into the
**File content / Preview** area.

You can also select a file and click **Open File**.

Note: the preview area is currently for viewing/editing text on screen, but
there is not yet a separate **Save File** button for saving edits made in that
preview box. To write file contents, use **New name**, **New file content**, and
**Write File**.

### Delete A File Or Folder

1. Select a file or folder.
2. Click **Delete File** or **Delete Directory**.
3. Click **Confirm Delete**.

By default, the app asks for confirmation before deleting.

## Configuration

On startup, the app creates `config.lst` if it does not already exist:

```text
safe_mode = true
confirm_delete = true
symbol_for_root = "R"
```

Settings:

- `safe_mode`: when `true`, deletion requires confirmation
- `confirm_delete`: when `true`, deletion requires confirmation
- `symbol_for_root`: text shown in the top bar beside the app title

## Safety Notes

- The file browser is designed to stay inside `environment/`.
- The **Back** button cannot navigate above `environment/`.
- Delete actions use a confirmation step by default.
- The `target/` directory should not be committed because Cargo can regenerate it.
- Files created by the app are ordinary files on disk, so only use the app in a
  workspace where you are comfortable creating, overwriting, and deleting files.

## Troubleshooting

If `target/debug/nexiscore_gui.exe` does not exist, build the project first:

```powershell
cargo build
```

If dependencies are missing or the build cache is stale, try:

```powershell
cargo clean
cargo build
```

If the app opens an unexpected `environment/` folder, close it and launch from
the repository root:

```powershell
cd path\to\nexiscore
cargo run
```
